## S in SOLID

S in SOLID stands for **Single Responsibility Principle**.

It means a class should have only one responsibility or one reason to change.

Example:

```python
class Invoice:
    def calculate_total(self):
        pass

class InvoicePrinter:
    def print_invoice(self):
        pass
```

Here, one class handles invoice logic and another handles printing.

## What is an App in Django?

A Django app is a module that handles one specific feature of a project.

Examples of apps:

- accounts
- posts
- chat
- stories
- groups

A Django project can contain multiple apps.

Example:

```bash
python manage.py startapp posts
```

If you create an app, you usually add it in `INSTALLED_APPS` in `settings.py`.

## I in ACID Property

I in ACID stands for **Isolation**.

ACID properties are used in databases and transactions.

Isolation means one transaction should not interfere with another transaction.

For example:

- two users updating the same record at the same time
- one transaction should not read half-completed data from another

ACID stands for:

- Atomicity
- Consistency
- Isolation
- Durability

## Difference Between Web Server and Application Server

### Web Server
A web server mainly handles:

- static files
- HTTP requests
- routing requests to the correct service

Examples:
- Nginx
- Apache

### Application Server
An application server runs the application logic.

Examples:
- Gunicorn
- uWSGI
- Django app process

In a Django setup:

- Nginx can act as web server
- Gunicorn can act as application server

## Difference Between `makemigrations` and `migrate`

### `makemigrations`
Creates migration files based on model changes.

Example:

```bash
python manage.py makemigrations
```

### `migrate`
Applies those migration files to the database.

Example:

```bash
python manage.py migrate
```

Simple understanding:

- `makemigrations` → create migration instructions
- `migrate` → execute them on database

## What is a Foreign Key in SQL?

A foreign key is a field in one table that refers to the primary key of another table.

It creates a relationship between two tables.

Example:

### Users table
| id | username |
|----|----------|
| 1  | abhay    |

### Posts table
| id | content | user_id |
|----|---------|---------|
| 1  | Hello   | 1       |

Here, `user_id` in `Posts` is a foreign key referring to `Users.id`.

In Django:

```python
author = models.ForeignKey(User, on_delete=models.CASCADE)
```

## How Can You Access an Element in a Django Template?

You can access variables in templates using double curly braces.

Example:

```html
{{ user.username }}
{{ post.content }}
```

You can also access dictionary values and object attributes.

Example:

```html
{{ profile.bio }}
```

In loops:

```html
{% for post in posts %}
    {{ post.content }}
{% endfor %}
```

## What is a Candidate Key?

A candidate key is a column, or combination of columns, that can uniquely identify a row in a table.

A table can have multiple candidate keys.

One of them is chosen as the primary key.

Example:

In a student table:
- `student_id`
- `email`

Both may uniquely identify a student, so both can be candidate keys.

## What is a Media File?

In web development, media files are files uploaded by users.

Examples:
- profile pictures
- post images
- videos
- documents

In Django, media files are usually configured using:

```python
MEDIA_URL = "/media/"
MEDIA_ROOT = BASE_DIR / "media"
```

Static files and media files are different:

- static files → created by developer
- media files → uploaded by users
