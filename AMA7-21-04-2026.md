## Different Types of Middleware

In Django, middleware is a framework of hooks that processes requests and responses globally.

Common middleware types in a Django project:

- Security middleware
- Session middleware
- Common middleware
- CSRF middleware
- Authentication middleware
- Message middleware
- Clickjacking middleware

Example from a Django project:

```python
MIDDLEWARE = [
    "django.middleware.security.SecurityMiddleware",
    "django.contrib.sessions.middleware.SessionMiddleware",
    "django.middleware.common.CommonMiddleware",
    "django.middleware.csrf.CsrfViewMiddleware",
    "django.contrib.auth.middleware.AuthenticationMiddleware",
    "django.contrib.messages.middleware.MessageMiddleware",
]
```

Middleware is useful because it lets you apply common logic to every request and response.

## What is the Use of `admin.py`?

`admin.py` is used to register models in the Django admin panel.

This helps you manage database records from the browser.

Example:

```python
from django.contrib import admin
from .models import Post

admin.site.register(Post)
```

After registering a model, you can log in to `/admin/` and manage posts, users, comments, and other data.

## What is HTTP Protocol?

HTTP stands for **HyperText Transfer Protocol**.

It is the protocol used for communication between a browser and a web server.

For example:

- Browser sends a request to open a page
- Server sends a response with HTML, JSON, images, or other data

Example:

```http
GET /home HTTP/1.1
Host: example.com
```

Common HTTP methods:

- `GET` → fetch data
- `POST` → send data
- `PUT` → update data
- `DELETE` → delete data

## How to Handle Password Fields

Passwords should never be stored as plain text.

Best practice is to:

- hash passwords
- validate password strength
- hide password input in forms

In Django, use built-in authentication tools.

Example:

```python
from django.contrib.auth.models import User

user = User.objects.create_user(username="abhay", password="StrongPass123")
```

This stores the password securely in hashed form.

In HTML:

```html
<input type="password" name="password">
```

## What is SMTP Protocol?

SMTP stands for **Simple Mail Transfer Protocol**.

It is used to send emails from one server to another.

Applications use SMTP for:

- password reset emails
- OTP emails
- welcome emails
- notifications

Example Django email settings:

```python
EMAIL_BACKEND = "django.core.mail.backends.smtp.EmailBackend"
EMAIL_HOST = "smtp.gmail.com"
EMAIL_PORT = 587
EMAIL_USE_TLS = True
```

## What are Closures?

A closure is a function that remembers variables from its outer scope even after the outer function has finished executing.

Example:

```python
def outer(x):
    def inner(y):
        return x + y
    return inner

add_five = outer(5)
print(add_five(3))
```

Output:

```python
8
```

Here, `inner()` remembers the value of `x`.

## How to Implement a User Model from Scratch

In Django, if you want a custom user model, it is better to create it at the start of the project.

A common way is to extend `AbstractUser`.

Example:

```python
from django.contrib.auth.models import AbstractUser
from django.db import models

class User(AbstractUser):
    bio = models.TextField(blank=True)
    profile_picture = models.ImageField(upload_to="profile_pics/", blank=True, null=True)
```

Then in `settings.py`:

```python
AUTH_USER_MODEL = "accounts.User"
```

This allows you to add extra fields like bio, date of birth, and profile picture.

## Syntax to Make a Website Responsive

A website becomes responsive mainly using:

- viewport meta tag
- flexible layouts
- media queries
- percentage widths
- Flexbox or Grid

Example:

```html
<meta name="viewport" content="width=device-width, initial-scale=1.0">
```

Example CSS:

```css
.container {
    display: flex;
    flex-wrap: wrap;
}

.card {
    width: 100%;
}

@media (min-width: 768px) {
    .card {
        width: 50%;
    }
}
```

This makes the layout adapt to screen size.

## Four Pillars of OOP

The four pillars of Object-Oriented Programming are:

- Encapsulation
- Abstraction
- Inheritance
- Polymorphism

### Encapsulation
Wrapping data and methods into a single unit.

### Abstraction
Showing only essential details and hiding internal implementation.

### Inheritance
One class can reuse properties and methods of another class.

### Polymorphism
The same method name can behave differently in different classes.

Example:

```python
class Animal:
    def sound(self):
        pass

class Dog(Animal):
    def sound(self):
        return "Bark"

class Cat(Animal):
    def sound(self):
        return "Meow"
```
